Github Pages

O GitHub é uma excelente saída para quem quer desenvolver um portfólio estático de seu trabalho, por exemplo, e deixar ele lá, grátis, com link customizável e, sem precisar de registro de domínio e hospedagem. No entanto, a estrutura de pastas deverá ser igual a da foto abaixo, pois, para um site estático, o GitHub necessita que todos os arquivos estejam em uma única pasta, tais como scripts em JavaScript, imagens, arquivos .php e folhas de estilo. Ou seja, se por ventura em seu código, tem algo do tipo: _css/style.css, ele deverá ser trocado por somente: style.css, sem invocação a pastas. Como subir um site no GitHub Pages? A primeira coisa necessária é ter uma conta criada em seu GitHub. Caso não a tenha, basta acessar github.com e criar a sua, de modo fácil e rápido. Para começar, devemos clicar em criar um repositório, clicando em New, conforme foto abaixo. Adicionar um título e uma descrição qualquer e o repositório deverá ser público. Observação: o título do repositório que irá aparecer no link de acesso ao site. Após isso, clicar em CREATE REPOSITORY, ao final da página. Depois, é necessário também ter o Git instalado em sua máquina. Caso não a tenha, só acessar: https://git-scm.com/, que fornecem o passo a passo para instalação dele. Após isso, basta clicar no botão direito de sua página que deseja subir no GitHub, já com os arquivos nela e clicar em GIT BASH HERE. Abrirá uma espécie de terminal a você. Para iniciar o git, deve-se digitar o comando git init e apertar Enter. Após isso, copiar o comando que o GitHub forneceu após a criação do remoto, para manter a sincronia do repositório com sua máquina, que é o git remote add origin…. e apertando Enter, digitar o comando git push -u origin master, para enviar os arquivos de uma vez ao repositório do GitHub. Se o processo for feito da forma correta, os arquivos aparecerão no seu repositório, após atualização da página. Para o GitHub Pages entrar em ação agora, clicar em SETTINGS, à direita de seu repositório. Quase ao final da página SETTINGS, em OPTIONS, aparecerá a opção do GitHub Pages para disponibilizar seu site nele. Para isso, você deverá: - Escolher a branch master; - O folder /docs; - Escolher um tema, caso seja preciso, mas é opcional; - Personalização de domínio a sua escolha, também opcional.
